## 既存Widgetでは「どうにもならない」瞬間

Flutterの標準Widgetは豊富で、多くのUIはそれらの組み合わせで実現できる。しかし開発を続けていると、「標準Widgetでは表現できないUI」に必ずぶつかる。

- グラフや波形など、データに基づいて動的に形が変わる図形
- 複雑なカスタムローダー、プログレスインジケーター
- ゲーム的なアニメーション表示
- デザイナーが描いたイラストに近い独自UI

こういったケースに対して、FlutterはCanvas APIを直接操作する仕組みを提供している。それが **`CustomPainter`** だ。

---

## CustomPainterの基本構造

`CustomPainter` はFlutterのCanvasに直接描画するための抽象クラスだ。`paint` メソッドにキャンバスへの描画ロジックを書き、`shouldRepaint` で再描画が必要かどうかを制御する。

```dart
class CircleProgressPainter extends CustomPainter {
  final double progress; // 0.0 〜 1.0

  CircleProgressPainter({required this.progress});

  @override
  void paint(Canvas canvas, Size size) {
    final center = Offset(size.width / 2, size.height / 2);
    final radius = size.width / 2;

    // 背景の円
    final backgroundPaint = Paint()
      ..color = Colors.grey.shade200
      ..style = PaintingStyle.stroke
      ..strokeWidth = 8.0;

    canvas.drawCircle(center, radius, backgroundPaint);

    // 進捗の弧
    final progressPaint = Paint()
      ..color = Colors.blue
      ..style = PaintingStyle.stroke
      ..strokeWidth = 8.0
      ..strokeCap = StrokeCap.round;

    canvas.drawArc(
      Rect.fromCircle(center: center, radius: radius),
      -math.pi / 2,           // 12時方向からスタート
      2 * math.pi * progress, // progressに応じた角度
      false,
      progressPaint,
    );
  }

  @override
  bool shouldRepaint(CircleProgressPainter oldDelegate) {
    return oldDelegate.progress != progress;
  }
}
```

使う側は `CustomPaint` Widgetで包む。

```dart
CustomPaint(
  size: const Size(100, 100),
  painter: CircleProgressPainter(progress: 0.75),
)
```

## `paint` メソッドで使える描画API

Canvas APIには豊富な描画メソッドがある。代表的なものを把握しておくと、表現の幅が広がる。

**図形描画：**
- `canvas.drawCircle` — 円
- `canvas.drawRect` — 矩形
- `canvas.drawRRect` — 角丸矩形
- `canvas.drawOval` — 楕円
- `canvas.drawLine` — 直線
- `canvas.drawArc` — 弧
- `canvas.drawPath` — パス（複雑な形状）

**テキスト描画：**
- `TextPainter` を使ってキャンバス上にテキストを配置できる

**変換：**
- `canvas.save()` / `canvas.restore()` でCanvasの状態を保存・復元
- `canvas.translate()` / `canvas.rotate()` / `canvas.scale()` で座標変換

**Path：**
最も柔軟なのが `Path` オブジェクトを使った描画だ。ベジェ曲線（`cubicTo`、`quadraticBezierTo`）を組み合わせることで、波形・吹き出し・カスタムシェイプを自由に描ける。

## `shouldRepaint` の設計が重要な理由

`shouldRepaint` は、**このパラメータが変わったときだけ再描画する**という制御の場所だ。ここを適切に実装しないと、無駄な再描画が走り続けてパフォーマンスが劣化する。

```dart
@override
bool shouldRepaint(CircleProgressPainter oldDelegate) {
  // progressが変わった時だけ再描画（色が変わっていない限り不要）
  return oldDelegate.progress != progress
      || oldDelegate.color != color;
}
```

逆に、常に `return true;` にしてしまうと、親Widgetが再ビルドされるたびに不要な再描画が発生する。**変更を正確に判断して最小限の再描画に留めること**が、CustomPainterのパフォーマンス設計の核心だ。

## アニメーションとの組み合わせ

CustomPainterはデータを受け取って描画するだけの存在なので、アニメーションとの組み合わせは非常に自然だ。`AnimationController` や `Tween` から値を受け取り、その値をCustomPainterに渡すだけでいい。

```dart
class _AnimatedProgressState extends State<AnimatedProgress>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _progressAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 1500),
      vsync: this,
    );

    _progressAnimation = Tween<double>(begin: 0.0, end: widget.targetProgress)
        .animate(CurvedAnimation(
          parent: _controller,
          curve: Curves.easeInOut,
        ));

    _controller.forward();
  }

  @override
  Widget build(BuildContext context) {
    return AnimatedBuilder(
      animation: _progressAnimation,
      builder: (context, child) {
        return CustomPaint(
          size: const Size(120, 120),
          painter: CircleProgressPainter(progress: _progressAnimation.value),
        );
      },
    );
  }
}
```

`AnimatedBuilder` がアニメーションの値が変わるたびに `CustomPaint` を再ビルドし、CustomPainterがその値で描画する。この分離のおかげで、ロジックが単純に保たれる。

## RepaintBoundaryで描画の影響範囲を制限する

CustomPainterはあくまでその描画領域内だけを再描画するが、親Widgetのビルドが走るとCustomPaintも再生成される。これを防ぐために `RepaintBoundary` を活用できる。

```dart
RepaintBoundary(
  child: CustomPaint(
    painter: MyComplexPainter(data: data),
    size: const Size(300, 300),
  ),
)
```

`RepaintBoundary` はその境界内の再描画を親から分離する。頻繁に更新されるアニメーションUIをRepaintBoundaryで囲むことで、残りのUIの再描画コストを節約できる。

## CustomPainterのテスト戦略

CustomPainterはUIの見た目に直結するため、**Golden Testと相性が良い**。期待する描画結果を画像ファイルとして保存し、以後の変更で見た目が変わっていないかを確認する。

```dart
testWidgets('CircleProgressPainter の描画が正しい', (tester) async {
  await tester.pumpWidget(
    MaterialApp(
      home: Scaffold(
        body: CustomPaint(
          size: const Size(100, 100),
          painter: CircleProgressPainter(progress: 0.5),
        ),
      ),
    ),
  );

  await expectLater(
    find.byType(CustomPaint),
    matchesGoldenFile('golden/circle_progress_0.5.png'),
  );
});
```

ただし、デバイスや環境によってアンチエイリアスやサブピクセルレンダリングに微妙な差が出ることがある。CI環境を固定し、一貫した環境でGolden Testを実行する運用が重要だ。

## どこまでCustomPainterを使うべきか

CustomPainterは強力だが、使いすぎに注意が必要だ。**標準Widgetで実現できることをCustomPainterで書くのは過剰**だし、CustomPainterで描いたUIはアクセシビリティの情報（Semantics）が自動では付かない。

カスタム描画を使うなら、`Semantics` Widgetで意味情報を別途付与する意識が必要だ。

CustomPainterを選ぶべき状況のサインは：
- データドリブンな図形（グラフ・チャート・波形）
- 標準Widgetのクリッピングや変形では実現できない形状
- フレームごとに細かい描画制御が必要なアニメーション

逆に、複雑な既存Widgetのスタイルカスタマイズや、Lottie/Riveで表現できるアニメーションは、そちらを使う方がメンテナンスしやすい。

---

## まとめ

FlutterのCustomPainterは、「Widgetの組み合わせ」という制約を超えて、Canvas APIを直接制御できる脱出ハッチだ。

大切なのは、`paint` メソッドでの描画ロジックよりも、**`shouldRepaint` によるパフォーマンス制御**と**AnimationControllerとの明確な役割分担**を意識することだ。描画の「どう見せるか」をCustomPainterに、「いつ変わるか」をアニメーション側に分離することで、複雑なUIでも見通しよく実装できる。

「このUIはWidgetで無理だな」と感じた瞬間が、CustomPainterを学ぶタイミングだ。
